Physical AI Lifelog Framework

Specification: 完全ローカル依存型バッテリーライフログアプリの仕様定義書

未来のフィジカルAIとの共生（関係性のチューニング）を事前学習・体験するための、完全ローカル依存型バッテリーライフログアプリの仕様定義書。

1. CSV Master Schema (データ構造)

アプリに同梱、およびユーザー間でインポート/エクスポートされるコアデータのヘッダー定義。

id,character_id,lifestyle_id,time_range,battery_level,charge_count,priority,message

2. 状態（State）の定義

Androidシステム（BatteryManager）の挙動と、フィジカルAIの生存（代謝・食事）のサイクルを紐付けるコアステート。

State ID システム条件（Android OSトリガー） 意味合い（世界観での定義） 備考（エッジケース処理）
IDLE BATTERY_STATUS_DISCHARGING または NOT_CHARGING 活動中（非充電状態）
エネルギーを消費しながら稼働している通常の佇まい。		 常駐サービス通知のベース状態。
CHARGING BATTERY_STATUS_CHARGING 食事開始（給電開始）
物理接続または非接触で通電が確定した瞬間。		 チャタリング防止のため、2秒間のデバウンス（遅延確定）後に移行。
DISCHARGING CHARGING から IDLE への移行 食事終了（給電停止）
満充電、または途中で引き抜かれた瞬間。		 抜線時の2秒デバウンス後に移行。
STARVED ACTION_SHUTDOWN 検知時かつ残量しきい値が「3%以下」の場合 餓死からの復帰（バッテリー切れ）
端末が完全に力尽きた後の再起動時に確定する状態。		 ※ユーザーの任意シャットダウン（残量15%以上など）の時は IDLE として処理する。

3. コンテキスト（動的センサー）の定義

time_range（時間帯）

CSVの値 Android側での判定条件（24時間表記） 意味合い
MORNING 05:00 以上 〜 10:59 以下 朝、1日の始まりの充電
DAY 11:00 以上 〜 17:59 以下 日中、仕事や外出中の継ぎ足し
NIGHT 18:00 以上 〜 22:59 以下 夜、帰宅後やリラックスタイム
MIDNIGHT 23:00 以上 〜 04:59 以下 深夜、夜更かしまたは就寝時の充電
* 条件なし すべての時間帯にマッチ

battery_level（バッテリー残量）

CSVの値 Android側での判定条件（％） 意味合い
LOW 20% 未満 空腹・危機的（OSのバッテリーセーバー推奨領域）
MID 20% 以上 〜 85% 未満 通常状態（適度なお腹の減り具合）
HIGH 85% 以上 満腹に近い（すでに十分蓄えがある状態）
* 条件なし すべての残量にマッチ

charge_count（1日あたりの充電回数）

CSVの値 Android側での判定条件（回数） 意味合い
1 1 回目 その日最初の食事（朝充電や、安定したリズム）
FEW 2 回 〜 3 回 日常的な範囲内での追加の食事
MANY 4 回 以上 頻繁な抜き差し（バッテリーのヘタリや忙しい日）
* 条件なし 何回目でもマッチ

4. ユーザー属性（ライフスタイル型）の定義

ユーザーの生活リズムをあらかじめ定義し、コンテキストに「意味」を与えるための属性。

属性ID ラベル（イメージ） 活動リズムの定義 ターゲット層
STANDARD 朝型 / 一般的なビジネスリズム 07:00〜23:00が主活動時間 カレンダー通りの会社員、学生、朝型の主婦など
NIGHT_OWL 夜型 / クリエイティブリズム 11:00〜03:00が主活動時間 夜型の学生、クリエイター、フリーランスなど
FLEXIBLE 不規則 / シフトワークリズム 24時間特定のコアを持たない シフト制勤務（医療・介護・警備）、ノマドなど
HOME_BASED 在宅 / インドアリズム 自宅中心で端末が常に近くにある リモートワーカー、インドア派生活の人など

5. 関係性テーマ（Relationship Themes）の定義

ユーザーがAIとの距離感を調律・配布・トレードするための「思想的ベースキャンバス」。

モードID ラベル（思想テーマ） 初期配布（Default CSV）の英語アプローチ ユーザーがこのテーマを「調律・共有」する意味
mirror_01 Mirror（自己延長 / 鏡） Charging confirmed.
主語を排した純粋なデータ・事実の記述。		 自身の生活リズムを、感情バイアスなしで「客観視・記録」するためのテーマ。
symbiosis_02 Symbiosis（共生生物 / 代謝） Energy absorbed.
AI側の「内部の渇きや満腹」を主軸にした表現。		 デバイスを「別の生命体」として扱い、観葉植物のようにそのサイクルをケアするテーマ。
infrastructure_03 Agent（自立インフラ / 契約） Power grid connected.
パフォーマンス、稼働率などのシステム用語。		 コックピットの管理AIのように、徹底して「機能的でドライな相棒」を構築するためのテーマ。
outsider_04 Observer（異邦人 / 観測） A quiet night begins.
時間、空間、人間の奇妙な習慣への言及。		 映画のSF AI（『HER』など）のような、少し詩的で他者性の強い世界観を表現するためのテーマ。
mediator_05 Tool（伴走仲介 / 道具） Got you. Recharging.
短く、人間と呼吸を合わせるような応答。		 長年連れ添った仕事道具のように、最も日常にノイズなく溶け込む距離感を追求するテーマ。

6. テキスト構造（Text Architecture）の定義

世界共通のフレームワークとしてスケールさせるため、英語をマスターデータ（基準）とし、ユーザーのオーバーライド（任意書き換え）を最優先する3層のレイヤー構造。

テキストデータ管理レイヤー

Layer 1: CSV Master (EN)
➔
Layer 2: Auto-Translate Cache (Local)
➔
Layer 3: User Override
  1. Layer 1（同梱CSV）: 英語で書かれた各テーマ×属性のマトリクス（初期コアメッセージ）。
  2. Layer 2（翻訳キャッシュ）: 初回起動時にLayer 1を端末言語（例: 日本語）に機械翻訳し、SharedPreferences へキャッシュ。
  3. Layer 3（ユーザー編集）: 設定画面からユーザーがメッセージを直接書き換えた場合、Layer 2を上書きして最優先で適用。すべての関係性テーマで自由編集が可能。上書きのリセットも可能。

結合システム（Token Insertion）の文法ルール

ユーザーが文章内の自由な位置に配置・編集できる変数トークン。

トークン 定義 ユーザー設定の例 出力メッセージの変化（日本語時）
{tag} ユーザーの呼び名（二人称） Master（マスター） ・{tag}, charging started. ➔ マスター、充電が始まりました。
・Charging started, {tag}. ➔ 充電が始まりました、マスター。
{suffix} 口調のニュアンス（接尾語） :)（ね） ・Power updated {suffix} ➔ 電力を更新したよ
・Power updated {suffix} ➔ 電力を更新しました :)
